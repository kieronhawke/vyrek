# Vyrek — Claude Code Execution Brief (v2)

**Scope:** Site-wide image overhaul + Quiz redesign + Results Hub (the big one) that matches hyresult.com feature-for-feature and beats it on UX, polish, data, and insight.

**Approach:** Read this entire brief before touching code. Plan the work, ask clarifying questions in one batch, then execute Parts 1 → 2 → 3 in order. Each Part has its own acceptance criteria.

---

## Global ground rules

- **Stack:** Next.js App Router + React + TypeScript + Tailwind + GSAP for motion. Confirm before deviating.
- **Never touch `<video>` elements** or their sources. Videos stay as-is.
- **Add new routes; don't break existing ones.**
- **Image discipline:** WebP, ≤1.5MB, ≤1920px long edge, EXIF orientation preserved, `next/image` everywhere with `alt` text and explicit dimensions.
- **No duplicate images** across the site unless intentional and commented.
- **Diversity audit** at the end of Part 1.
- **Every commit** references the Part/section it advances.
- **Never invent or AI-generate imagery.** Use only the photos in the project's `photos` folder. Where no match exists, leave the existing image or use a neutral logomark placeholder.
- **Never copy text, code, or assets from hyresult.com.** Their feature set is the reference; copy and visual design are entirely Vyrek's.

# PART 1 — Photo folder audit & site-wide image replacement

## 1.1 Locate and inventory the folder

The user has added a `photos` folder somewhere in the project (likely `/photos`, `/public/photos`, `/assets/photos`, or `/public/media/photos`). Find it. For every file, capture: filename, dimensions, file size, format, EXIF date if present, and a content description generated by inspecting the image. Inspection should yield:
- Subjects (gender, approx age, ethnicity, count)
- Setting (gym, outdoor, studio, race venue, specific station)
- Mood (intense, joyful, exhausted, focused, candid)
- Composition (portrait/landscape, headroom, subject placement)
- Lighting (bright daylight, moody warehouse, golden hour, harsh fluorescent)
- Suitability tags: `hero`, `coach-portrait`, `programme-card`, `bento`, `testimonial`, `quiz-interstitial`, `results-banner`, `about-page`, `inline-blog`, `og-image`, `event-card`, `athlete-headshot`

Save to `docs/photo-inventory.json` with stable IDs.

## 1.2 Audit every current image on the site

Walk every page and component. For each `<Image>`, `<img>`, or `background-image`, record file path, usage location, role, and verdict: **keep / replace / remove**. Replace = AI-looking, generic, low-res, blurry, off-brand, duplicated. Remove = slot better empty or restructured. Save to `docs/image-audit.md` and show me before replacing.

## 1.3 Match photos to slots

For each "replace" slot, pick the best photo by **semantic fit** (coach portrait → portrait photo, sled-station card → sled photo, doubles programme → two athletes), **composition fit** (hero needs landscape with text headroom, bento needs balanced focal points, testimonial needs tight crop), and **mood fit** (Vyrek is gritty, serious, aspirational — not stocky or smiley unless slot calls for warmth). Document matches in `docs/image-placement-map.md`: slot → photo ID → one-sentence rationale.

## 1.4 Replace, crop, optimise

Crop to slot aspect ratio (never stretch). Convert to WebP. JPEG fallback only when component needs it. Generate `@1x` and `@2x` for responsive slots. Output to `/public/media/images/[category]/[descriptive-filename].webp`. Update components to new paths with `alt` text describing what's in the photo.

## 1.5 Manifest and diversity audit

Produce `docs/image-manifest.json`: every in-use image with path, used-on pages, subject tags, dimensions, file size. Then a diversity audit: gender, age bracket, ethnicity, body type counts → `docs/diversity-report.md`. Rebalance if any category is severely under-represented.

## 1.6 No-match handling

If no folder image matches a slot, leave the current image if acceptable or replace with a neutral dark-gradient + Vyrek logomark placeholder. Log every gap in `docs/missing-photos.md`.

## 1.7 Acceptance criteria

- [ ] All images audited; kept/replaced/marked missing
- [ ] No duplicate image in two roles
- [ ] All replaced images WebP, ≤1.5MB, with alt text
- [ ] Diversity report shows balanced representation
- [ ] Lighthouse ≥90 Performance on every changed page

## 3.3 Data sourcing (decision still open)

**No public HYROX results API exists.** Options:

**A. Scrape and store** — pull from `results.hyrox.com` (SAP/mika:Timing) and event pages on `hyrox.com`. Scheduled via Vercel Cron. Normalise into Postgres / Supabase. Re-scrape every 5 minutes during live event weekends, daily otherwise. Respectful rate limits (1 req/sec), identifying user-agent. **Legal grey area — confirm with me before deploying.**

**B. Manual seed JSON** — curate latest 2–3 major events into committed JSON. Ship UI fast against this. Full pipeline in v2.

**C. Pursue official partner feed** — slowest, best long-term.

**Recommendation:** Start with B to ship UI immediately. Build A's scraper in parallel, deploy only with my OK. Always show a `STATUS` chip in dev: "Seed data, last updated YYYY-MM-DD" vs "Live, updated 30s ago".

## 3.4 Time-gated funnel (anti-bypass implementation)

**Behaviour:**
- First 5:00 minutes per session on `/results/*` → full access, no friction
- At 5:00 → soft slide-up modal: "Continue analysing higher-up data — sign up to keep going."
  - Primary CTA: "Create free account" → signup
  - Secondary CTA: "Maybe later" (snooze, 60s, one-time only)
- After snooze expires or modal dismissed without signup → results pages render with **top half visible, bottom half blurred** behind a sign-up wall
- Logged-in free accounts → unlimited Results access
- Paid course members → unlock **Pro analytics**: radar overlays, predictive simulator, custom comparisons, coach commentary, race recap deep insights, OG card customisation

**Anti-bypass:**
- Session counter server-side in Vercel KV / Upstash Redis (key by user agent + IP hash + cookie ID, with TTL 24h)
- localStorage acts only as a UX hint, not the source of truth
- Incognito and cookie-clearing will bypass — accept this, don't over-engineer
- Skip the gate for crawlers (UA contains googlebot, bingbot, etc.) — SEO benefit
- Track ratio of fresh-cookie sessions as an abuse signal in analytics

**Telemetry:**
- Page reached when gate triggered
- Time on Results pages before gate
- Snooze used? Y/N
- Action after gate: signup / bounce / signup-later-in-session
- Conversion rate to course purchase among gated signups

## 3.6 Design system for Results pages

- **Theme:** dark by default, background `#0A0A0B`, subtle radial gradients in hero zones. Light mode optional, ship dark-only.
- **Accent palette:**
  - Primary (Vyrek brand) — for active states, primary actions
  - Electric cyan `#22D3EE` — LIVE indicators, real-time pulses
  - Magenta `#E879F9` — compare/versus elements
  - Amber `#FBBF24` — podium/leader highlights
  - Soft white `#F8FAFC` — primary text
  - Slate-400 `#94A3B8` — secondary text
- **Typography:**
  - Display headings — existing Vyrek font
  - Body — existing
  - **Tabular numerals everywhere** for tables and time displays: `font-feature-settings: 'tnum'`
  - Fixed-width style for live timer/clock displays to prevent digit jitter
- **Cards:** `bg-white/[0.03] backdrop-blur-xl border border-white/[0.06] rounded-2xl` with soft inset shadow
- **Tables:**
  - Sticky header on scroll
  - Zebra rows at 0.5% white
  - Hover row at 2% white
  - Mobile: collapse to stacked cards with key fields up top
  - Virtualised for >100 rows
- **Charts:** custom Recharts dark theme file. Always show data freshness in chart footnote.
- **Live indicator:** cyan dot 8px, pulsing 1.5s, with "LIVE" label and last-update timestamp
- **Number reveals:** counts and times animate from 0 to value, 600ms easeOut, on first paint and on data change
- **Skeleton states:** every async block has a skeleton, never a spinner
- **Country flags:** SVG, sourced from a stable CDN or self-hosted; support regional variants (gb-eng, gb-sct, gb-wls)
- **Empty states:** every list has a custom empty state with illustrative photo from new inventory + helpful CTA
